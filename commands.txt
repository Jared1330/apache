Apache состоит из ядра и модулей (лежат в /usr/lib/apache2/modules)

По умолчанию в Apache 2.4 подключен модуль prefork
Prefork - каждый запрос обслуживается отдельным процессом Apache. Основное преимущество prefork - стабильность, 
процессы изолированы друг от друга и ошибки при обработке одного запроса ни как не затронут другие запросы.

Worker - сочетает в себе мульти-процессовый (multi-process) и мульти-поточный (multi-threaded) сервер

Установка
sudo apt update
sudo apt install apache2
sudo systemctl enable apache2

Конфигурационные файлы:
/etc/apache2/apache2.conf - основные настройки
/etc/apache2/conf-available/* - дополнительные настройки веб-сервера
/etc/apache2/mods-available/* - настройки модулей
/etc/apache2/sites-available/* - настойки виртуальных хостов
/etc/apache2/ports.conf - порты, на которых работает веб сервер
/etc/apache2/envvars - переменные, которые можно использовать в других конфигурационных файлах

Параметры основного файла конфигурации /etc/apache2/apache2.conf:
Timeout - указывает как долго сервер будет пытаться продолжить прерванную передачу или прием данных.
KeepAlive - позволяет передавать несколько файлов, за одно соединение, например, не только саму html страницу, но и картинки и css файлы.
MaxKeepAliveRequests - максимальное количество запросов за одно соединение, чем больше, тем лучше.
KeepAliveTimeout - таймаут соединения, обычно для загрузки страницы достаточно 10-20 секунд.
User, Group - пользователь и группа, от имени которых будет работать программа.
HostnameLookups - записывать в логи вместо ip адресов доменные имена, лучше отключить, чтобы ускорить работу.
LogLevel - уровень логирования ошибок. По умолчанию используется warn
Include - все директивы include отвечают за подключение дополнительных конфигурационных файлов.


Модули - расширяют функциональность Apache. 
Модули находяться в  /etc/apache/mods-available. А активированные в /etc/apache/mods-enable

Посмотреть все запущенные модули можно командой:
apache2ctl -M
Включить модуль: sudo a2enmod имя_модуля
Отключить: sudo a2dismod имя_модуля


Виртуальный хостинг - применяется для полноценного обслуживания сразу нескольких различных сайтов на одном физическом сервере.
Он обеспечивает независимую друг от друга работу нескольких сайтов на одном IP адресе в рамках одного физического сервера.

Для создания нового хоста достаточно создать файл с именем хоста и заполнить его нужными данными. 
Обернуть все эти параметры нужно в директиву VirtualHost
ServerName - основное имя домена
ServerAlias - дополнительное имя, по которому будет доступен сайт
ServerAdmin - электронная почта администратора
DocumentRoot - папка с документами для этого домена

Пример:
/etc/apache2/sites-available/actpro.site.conf
<VirtualHost *:80>
ServerName actpro.site
ServerAlias www.actpro.site
ServerAdmin webmaster@localhost
DocumentRoot /var/www/actpro.site/index.html
ErrorLog ${APACHE_LOG_DIR}/error.log
CustomLog ${APACHE_LOG_DIR}/access.log combined
</VirtualHost>

Виртуальные хосты, как и модули нужно активировать:
sudo a2ensite actpro.site
Для отключения:
sudo a2dissite actpro.site
